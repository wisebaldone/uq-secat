API

Methodology

The project was always meant to be a statically served single page web app so the api had to rely on served json files. To search for a course's stats you first load up the courses.json which contains a mapping of each available course to the semesters and years that it was run. With this information you can then get the data by using a http get of the form api/COURSECODE/YEAR/SEM.json.

Endpoints

wisebaldone.github.io/uq-secat/api/courses.json

The courses endpoint is a little confusing to explain and is easier seen. Each course is in a dictionary that contains a dictionary of the years which contain arrays of sems available.
